# SyncFlow Mobile i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add zh/en multi-language support to `apps/mobile` using `react-i18next` + `react-native-localize`, following the system locale only (no manual switch), as specified in `docs/superpowers/specs/2026-04-17-mobile-i18n-design.md`.

**Architecture:** Single i18next `translation` namespace with nested JSON groups per screen. `AppError` class carries i18n keys as error codes. Locale resolved once at boot; non-`zh-Hans` falls back to English. No runtime hot-switch.

**Tech Stack:** React Native 0.84 + React 19.2 · `i18next@^23` · `react-i18next@^15` · `react-native-localize@^3` · Jest + `react-native` preset · TypeScript strict mode.

1. **Existing Chinese text is mixed simplified and traditional.** Example: `apps/mobile/src/screens/LoginScreen.tsx` has both `请输入正确的手机号码` (simplified) and `手機號碼格式不正確` (traditional). **Extract verbatim — do NOT normalize.** `zh.json` preserves whatever was on screen. 2. **`services/auth-service.ts`, `services/api.ts`, `services/subscription-service.ts`, `utils/phone-validation.ts` contain no Chinese** — P5 work is mostly in screens' Alert messages, not in service throws.
3. **Jest preset is `react-native`** — `@testing-library/react-native` is not yet installed in `apps/mobile/package.json`. Task 7 adds it when we introduce the first component-level i18n test.
4. **All work happens in `apps/mobile/`.** Do not modify `apps/desktop`, `packages/contracts`, `packages/design-tokens`, or the Go sidecar.

## Migration Pattern (applies to every screen task)

**Before:**
```tsx
<Text style={styles.title}>设置</Text>
<Button title="退出登录" onPress={handleLogout} />
Alert.alert('错误', '发送失败，请稍后重试');
```

**After:**
```tsx
import { useTranslation } from 'react-i18next';

export function SettingsScreen() {
  const { t } = useTranslation();
  // ...
  return (
    <>
      <Text style={styles.title}>{t('settings.title')}</Text>
      <Button title={t('settings.logout')} onPress={handleLogout} />
      {/* Alert inside handler */}
    </>
  );
}

// inside handler
Alert.alert(t('errors.title'), t('errors.sendFailed'));
```

**Rules:**
- One `const { t } = useTranslation()` per component function body.
- No string concatenation — use `{{param}}` interpolation: `t('settings.versionLabel', { version, build })`.
- Keys are added to **both** `zh.json` (existing Chinese verbatim) **and** `en.json` (English translation) in the same commit.
- If a string is exactly `'v' + version + ' (' + build + ')'`-like, the translation key contains the literal format: `"versionLabel": "v{{version}} ({{build}})"`.

- [ ] **Step 3: Implement resolveLocale**

Create `apps/mobile/src/i18n/locale-resolver.ts`:
```ts
import type { Locale } from 'react-native-localize';

export type SupportedLocale = 'zh' | 'en';

/**
 * Picks the first locale in the user's preference list that matches a language
 * we support. Simplified Chinese (zh-Hans or bare zh) -> 'zh'. Everything else
 * (including Traditional Chinese) -> 'en'.
 */
export function resolveLocale(locales: readonly Locale[]): SupportedLocale {
  for (const l of locales) {
    if (l.languageCode === 'zh' && l.scriptCode !== 'Hant') {
      return 'zh';
    }
    if (l.languageCode === 'en') {
      return 'en';
    }
  }
  return 'en';
}
```

Note the loop returns `en` as soon as it sees English — so a Traditional Chinese user whose preference list also contains English will land on English rather than scanning further.

    en: { translation: en },
  },
  lng,
  fallbackLng: 'en',
  interpolation: { escapeValue: false },
  returnNull: false,
  missingInterpolationHandler: __DEV__

## Task 17: CJK residual scanner script

**Files:**
- Create: `apps/mobile/scripts/extract-cjk.mjs`

- [ ] **Step 1: Write the script**

Create `apps/mobile/scripts/extract-cjk.mjs`:
```js
#!/usr/bin/env node
// Scans apps/mobile/src for Chinese residues outside of locale JSONs.
// Usage: node apps/mobile/scripts/extract-cjk.mjs

import { readdir, readFile, stat } from 'node:fs/promises';
import { join, relative } from 'node:path';

const ROOT = new URL('../src/', import.meta.url).pathname;
const EXCLUDE_DIRS = new Set(['__tests__', '__snapshots__']);
const EXCLUDE_FILES = new Set(['zh.json', 'en.json']);
const EXTENSIONS = new Set(['.ts', '.tsx', '.js', '.jsx']);
const HAN = /\p{Script=Han}/u;

async function walk(dir, acc = []) {
  for (const entry of await readdir(dir, { withFileTypes: true })) {
    if (entry.isDirectory()) {
      if (EXCLUDE_DIRS.has(entry.name)) continue;
      await walk(join(dir, entry.name), acc);
    } else if (entry.isFile()) {
      if (EXCLUDE_FILES.has(entry.name)) continue;
      if (!EXTENSIONS.has(entry.name.slice(entry.name.lastIndexOf('.')))) continue;
      acc.push(join(dir, entry.name));
    }
  }
  return acc;
}

const files = await walk(ROOT);
let hits = 0;
for (const file of files) {
  const text = await readFile(file, 'utf8');
  const lines = text.split('\n');
  for (let i = 0; i < lines.length; i++) {
    if (HAN.test(lines[i])) {
      console.log(`${relative(process.cwd(), file)}:${i + 1}:${lines[i].trim()}`);
      hits++;
    }
  }
}

if (hits > 0) {
  console.error(`\nFound ${hits} residual CJK line(s). Migrate them via i18n keys.`);
  process.exit(1);
} else {
  console.log('Clean — no residual CJK outside locale files.');
}
```
